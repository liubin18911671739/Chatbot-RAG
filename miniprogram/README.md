# 微信小程序开发配置文件

## 项目结构

```
miniprogram/
├── app.js                 # 小程序主入口
├── app.json              # 小程序全局配置
├── app.wxss              # 小程序全局样式
├── sitemap.json          # 站点地图配置
├── pages/                # 页面目录
│   ├── index/           # 首页
│   ├── chat/            # 聊天页面
│   ├── scenes/          # 场景选择页面
│   ├── history/         # 历史记录页面
│   ├── profile/         # 个人资料页面
│   └── feedback/        # 反馈页面
├── utils/               # 工具类
│   ├── api.js          # API 服务
│   ├── storage.js      # 存储管理
│   └── utils.js        # 通用工具
└── images/             # 图片资源
    ├── logo.png        # 应用logo
    ├── ai-avatar.png   # AI头像
    ├── home.png        # 首页图标
    ├── scenes.png      # 场景图标
    ├── history.png     # 历史图标
    └── profile.png     # 个人图标
```

## 开发说明

### 1. API 配置
- 在 `utils/api.js` 中配置后端 API 地址
- 默认指向 `http://10.10.15.211:5000/api`
- 可根据环境调整 API 地址

### 2. 页面功能
- **首页**: 欢迎信息、快速问题、最近聊天
- **聊天页面**: 与AI助手对话
- **场景选择**: 选择不同的专业服务场景
- **历史记录**: 查看和管理聊天历史
- **个人资料**: 用户设置和应用管理
- **反馈页面**: 提交意见和建议

### 3. 存储管理
- 使用微信小程序的本地存储
- 自动保存聊天历史
- 支持用户设置持久化

### 4. 网络请求
- 统一的 API 服务封装
- 错误处理和重试机制
- 网络状态检测

### 5. 用户体验
- 响应式设计适配不同屏幕
- 加载状态提示
- 错误信息友好展示
- 分享功能支持

## 部署步骤

1. 在微信开发者工具中导入项目
2. 配置 AppID
3. 调整 API 地址配置
4. 测试各项功能
5. 提交审核发布

## 注意事项

- 需要在微信公众平台配置服务器域名
- API 服务器需要支持 HTTPS
- 图片资源需要放在 images 目录下
- 遵循微信小程序开发规范
